# Gemini 追撃プロンプト Phase 2

以下をそのまま Gemini に渡してください。

```text
あなたはインディーゲームのシニア・ナラティブデザイナー兼システムデザイナーです。
私は「群馬県を異界化したピクセルアートのロードムービーRPG」を作っています。

現在の固定仕様は以下です。

- 6章 + 終章
- 核となる感情: 土地への敬意 / 切ない旅 / 理解して通してもらう体験
- 戦闘: ダイスを使った儀式
- プレイ時間: 5〜8時間
- 一本道の推進力を最優先
- ランダムエンカウントとEXPレベルアップは削る方向
- 既存の戦闘別紙として `docs/gunma_ritual_battle_spec.md` がある

章別青写真は以下です。

- 第1章: だるま。HPを削ったあと「目を入れる」で終える
- 第2章: 製糸場。低い出目で糸をほどく
- 第3章: 草津。温度を下げて浄化する
- 第4章: 赤城・榛名。自分のダイスを譲渡して空虚を満たす
- 第5章: 水上・谷川・八ッ場。自傷で共鳴ダイスを作る
- 第6章: 尾瀬。過去5章のルールを切り替えて使う総復習
- 終章: 重要アイテムを奉納する

今回ほしいのは、既存の `docs/gunma_ritual_battle_spec.md` を踏まえて、さらに不足を埋める「実装向けの戦闘仕様書」です。

以下を日本語で出してください。

1. 各章ボス戦のUI仕様
   - どんなゲージを見せるか
   - 何を隠し、どの瞬間に開示するか
   - プレイヤーが誤解しやすい点

2. 各章ボス戦のチュートリアル導線
   - 戦闘前に何を見せるか
   - 戦闘中にどう学ばせるか
   - 失敗しても理解が深まる演出

3. 章ごとの「儀式失敗時のフォロー」
   - ただのゲームオーバーにしない
   - ナラティブと一致するやり直し方を提案する

4. 第6章の総復習ボスの形態変化案を3パターン
   - 形態名
   - 対応する過去章ルール
   - その形態が象徴する感情

5. 終章の奉納UIの設計
   - 何スロット必要か
   - 誤配置時にどうフィードバックするか
   - 最後に「ありがとう」をどう操作で感じさせるか

ルール:
- 一般論は少なめ
- 必ず「プレイヤーが何を理解するか」を中心に書く
- UIと感情を分離しない
- ピクセルRPG / Vanilla JS / 2D / 短編インディー前提で考える
```
