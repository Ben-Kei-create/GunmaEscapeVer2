# ChatGPT 追撃プロンプト Phase 2

以下をそのまま ChatGPT に渡してください。

```text
あなたはシニアゲームデザイナー兼、Vanilla JavaScript の実装補助エンジニアです。
私はブラウザ向けのピクセルRPGを作っています。

プロジェクトの固定仕様:

- 6章 + 終章
- コア感情: 土地への敬意 / 切ない旅 / 理解して通してもらう体験
- 戦闘はダイス儀式
- 一本道のロードムービー構造
- ランダムエンカウント削除
- EXPレベルアップ削除

章ごとの主要儀式:

- 第1章: HPを削ったあと「目を入れる」で終える
- 第2章: 低い出目で糸をほどく
- 第3章: 温度メーターを下げて浄化する
- 第4章: 自分のダイスを譲渡して空虚を満たす
- 第5章: 自傷で共鳴ダイスを作る
- 第6章: 過去の章ルールを切り替えて使う総力戦
- 終章: 重要アイテムを儀式陣に配置する

コードベースの前提:

- `js/story.js` に `journeyState`, `respectGauge`, `catalysts` は実装済み
- `js/items.js` に主要キーアイテムの `isCatalyst` 追加済み
- `js/player.js` で触媒の自動登録済み
- `js/save.js` は `journeyState` 保存対応済み
- `js/ui.js` には敬意ゲージ表示あり
- これから触る本命は `js/event.js`, `js/battle.js`, `js/ui.js`
- 戦闘別紙として `docs/gunma_ritual_battle_spec.md` がある

今回ほしいのは、`docs/gunma_ritual_battle_spec.md` を実コードへ落とすための「差分設計」と「疑似コード」です。

以下を出してください。

1. `battle.js` を章固有ルールへ拡張する設計
   - 共通戦闘ループと章固有フックをどう分けるか
   - ハードコード地獄を避けるためのデータ設計

2. 第1章〜第3章までの具体的なボス戦データ例
   - 必要な enemy パラメータ
   - 必要な battle state
   - 勝利条件 / 失敗条件

3. `ui.js` に必要な追加描画関数の一覧
   - 温度ゲージ
   - 絡まりゲージ
   - 奉納スロット
   - 空虚スロット

4. `event.js` 側で必要なイベントID案
   - 各章の導入
   - 儀式チュートリアル
   - ボス後の静かな余韻

5. 最後に
   - 第1スプリントで着手すべき順番
   - 第2スプリントで着手すべき順番
   - 第3スプリントで着手すべき順番

出力ルール:
- 日本語
- ふわっとした感想不要
- 擬似コードはそのまま JS に寄せられる粒度で
- 既に実装済みの `journeyState / respectGauge / catalysts` を前提にする
- 一本道の推進力と感情ピークを壊さない
```
