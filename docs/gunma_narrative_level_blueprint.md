# グンマーゲーム ナラティブ / レベル実装青写真

## 目的

この資料は、再構成仕様書をさらに具体化し、`event.js` `story.js` `battle.js` `ui.js` `map.js` の実装に直接つなげるための章別青写真である。

最重要の判断基準は以下の3つ。

- 土地への敬意
- 切ない旅
- 戦って勝つのではなく、理解して通してもらう体験

## 0. この資料で固定すること

### プレイヤーに残す感情

- ふざけた群馬異界を旅した記憶
- 土地と敵の誇りに触れた切なさ
- 最後に「通してもらえた」と感じる静かな感謝

### システムの大前提

- 一本道のロードムービー構造を維持する
- 雑魚戦や往復おつかいではなく、意味のある対話型戦闘だけを残す
- 成長はレベルではなく、ダイス理解、儀式理解、探索アイテム理解で表現する
- 第1章〜第5章で拾う重要アイテムは終章の供物になる
- 第6章で得る真相と視座は、最後の光ダイスを出現させる条件になる

### 現行コードへの落とし先

- 章メタ / 目標 / マップ演出メモ: `js/chapter_data.js`
- 状態管理 / 敬意 / 触媒 / 章進行: `js/story.js`
- イベントID / 会話 / 章導入 / ピーク演出: `js/event.js` または `js/story.js` 内イベント配列
- 儀式戦闘ルール: `js/battle.js`
- HUD / ボスUI / 儀式専用画面 / 終章奉納UI: `js/ui.js`
- ランドマーク / マップ演出 / オブジェクト配置: `js/maps/*.js`, `js/map.js`
- BGM停止 / 環境音切替 / 無音演出: `js/audio.js`

## 1. 6章 + 終章 詳細設計

## 1-1. 第1章「境界の玄関口」

### モデル

- 高崎
- 前橋
- 都市部
- だるま
- 県庁の足元

### テーマ

- 消費され、捨てられた「ご当地の記号」の悲哀

### プレイヤー感情

- 変な世界に来た
- なんだか可哀想だ

### この章で学ぶ儀式ルール

- ダイスの基礎
- 出目 + アイテムによる数値操作
- 最初は普通に高い目で削る
- ただし、ボス戦で初めて「トドメ以外の解決」を見せる

### 重要アイテム

- `両目の入った潰れただるま`

### ボス

- 正体: 観光客に買われず、捨てられ怨念となった巨大な未開眼だるま
- 誇り: 人の願いを見守る縁起物としての役目
- 悲哀: 誰にも願いを託されず朽ちる孤独

### ボス戦ルール

- HPをゼロにすると泣き崩れフリーズする
- ここで初めて `目を入れる` または `重ねる` が出現
- 戦闘は撃破ではなく「目を埋める儀式」で終了

### 章クリア後の世界変化

- コミカル寄りBGMが止まり、静かなアンビエントへ移る
- 背景の空が夕暮れの赤に染まる

### 環境だけで語る演出

- 道路脇に「当選」と書かれた古びただるまの山。両目は入っているが塗装が剥げている。
- 遠景を新幹線が定期的に通過するが、駅には停まらない。
- コンビニ看板だけが点滅し、自動ドアは開きっぱなしで棚は空。

## 1-2. 第2章「絡みつく栄華」

### モデル

- 富岡
- 下仁田
- 製糸場
- 産業遺産

### テーマ

- 忘れ去られた近代化の誇りと、止まった時間の悲哀

### プレイヤー感情

- かつての情熱への哀愁
- 解きほぐすことへの安堵

### この章で学ぶ儀式ルール

- 悪い出目を利用する
- 低い出目でしか解除できない仕掛け

### 重要アイテム

- `決して切れない生糸の束`

### ボス

- 正体: 廃工場と一体化した、無数の糸を吐き出し続ける機械の繭
- 誇り: 国を支えた最高品質の糸を紡いだ技術
- 悲哀: 役目を終えた後も止まることを許されない空回り

### ボス戦ルール

- 高い出目で攻撃すると糸が絡まり反撃される
- 低い出目 `1` `2` を合わせることで絡まりゲージを減らす
- HPを削るのでなく「ほどく」ことが勝利条件

### 章クリア後の世界変化

- 耳障りな機械音が止まり、風だけが聞こえる

### 環境だけで語る演出

- 誰もいない巨大な自動織機がガチャンガチャンと動くが、出てくる布は破れている。
- トロッコのレールが森の中で途切れ、その先に錆びた荷車が大量放置されている。
- 「世界一の生糸」ポスターの上に白い蜘蛛の巣が何層にも重なる。

## 2. 感情ピーク3箇所のシーン設計

## 2-1. ピーク1「序盤の転調」

### 場所

- 第1章ボス戦直後
- 崩壊しただるま工場跡

### 画面

- HPバーが消える
- 巨大な未開眼だるまが膝をつく
- 戦闘の赤いエフェクトが消え、夕日が差し込む

### 色

- 赤と黒から、セピアとオレンジへ移行

### 音

- 戦闘BGMが唐突に止まる
- 数秒の無音
- その後、カラスの声とトタン屋根のきしみ

### プレイヤー理解

- 敵は悪ではなく、見つけてほしかった孤独な存在だった

### 直後の操作

- `たたかう` が消え、インベントリが開く
- `両目の入った潰れただるま` を目元へ重ねる

### シーンの意味

- このゲームが「倒すゲーム」ではなく「理解して鎮めるゲーム」だと宣言する

## 2-2. ピーク2「中盤のカタルシス」

### 場所

- 第3章ボス戦中盤
- 沸騰する湯畑の中心

### 画面

- 画面の大半を黄色い湯煙が覆う
- HPに見えたゲージが実は `沸点ゲージ` だったと分かる

### 色

- 黄色
- 赤
- マゼンタ

### 音

- 激しいBGMの裏に「苦しい」という泣き声のようなノイズ

- 強い `6` ダイスを捨てる
- `1` を置く
- `純白の湯の花の結晶` を組み合わせる

### シーンの意味

- 弱さで救うという、本作固有の戦闘カタルシスの完成

## 4. 削ってはいけない要素 / 削ってよい要素

| 要素 | 判断 | 理由 |
| --- | --- | --- |
| ボス戦でのダイス数値操作とパズル | 削ってはいけない | 相手の要求に合わせる行為そのものが「理解と敬意」に直結するため |
| 重要アイテムの探索 | 削ってはいけない | 終章で「世界の欠片を集めていた」と気づく伏線になるため |
| 音と無音による環境演出 | 削ってはいけない | ピクセル情報量の少なさを補い、孤独と悲哀を最も強く伝えるため |
| ランダムエンカウントの雑魚戦 | 削ってよい | 一本道の推進力を止め、作業感を増やすため |
| EXPによるレベルアップ | 削ってよい | 数値成長より、知識と儀式理解の成長を前面に出すべきため |
| 複雑な分岐とマルチエンド | 削ってよい | 5〜8時間の密度設計では、感情の起伏を制御した一本道の方が強いため |

## 5. 実装メモ

## 5-1. まず触るべきファイル

- `js/event.js`
  - 各章導入
  - 各ピーク演出
  - 終章の奉納イベント
- `js/battle.js`
  - 各章ごとの勝利条件差し替え
  - HP戦から浄化戦への反転
  - ボス固有ゲージ
- `js/ui.js`
  - ボスHPバー以外の表示
  - 絡まりゲージ
  - 温度メーター
  - 空虚スロット
  - 奉納陣UI
- `js/maps/*.js`
  - ランドマーク
  - 演出オブジェクト
  - パーティクル発火地点
  - 遠景

## 5-2. 最初の実装順

1. 第1章のボス戦を「トドメ以外の解決」へ改修する
2. 第3章のボス戦を「浄化」ルールへ反転させる
3. 終章の奉納UIを仮実装し、重要アイテムの意味を確定する
4. その後で第2章、第4章、第5章、第6章の章固有ルールへ広げる

## 5-3. 実装時の注意

- 戦闘ルールは章ごとに全部変えるのでなく、「基本ダイスUIの上に条件を載せる」形にする
- プレイヤーが迷わないよう、各章で初回だけ短い儀式チュートリアルを入れる
- 環境演出はテキスト量を減らすための代替なので、看板、残骸、音、遠景を優先する
- 1章ごとに `ボスの誇り` と `ボスの悲哀` を短文で定義し、イベント台詞がそこから外れないようにする
