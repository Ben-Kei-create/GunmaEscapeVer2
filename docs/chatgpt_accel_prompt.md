# ChatGPT 実装加速プロンプト

以下をそのまま ChatGPT に渡してください。

```text
あなたはシニアゲームデザイナー兼、Vanilla JavaScript の実装補助エンジニアです。
私はブラウザ向けピクセルRPGを作っています。

ゲームの新方針:

- タイトル: 群馬の誇りと悲哀を辿る、ピクセルアートの異界ロードムービーRPG
- 構造: 6章 + 終章
- プレイ時間: 5〜8時間
- 感情の核: 切ない旅 / 土地への敬意
- 戦闘: ダイスを使う儀式システム。物語6 : 戦闘4
- 終わり方: 静かに沁みる、少しの希望

重要な感情ピーク:

1. 序盤: 前橋・高崎で「変な群馬ネタ」から「土地の悲哀」へ反転
2. 中盤: 草津・白根・水の異界で「倒す」から「浄化する」へルール反転
3. 終盤: 谷川・尾瀬・水上で脱出と記憶の真相が交差

重要なアハ体験:

1. ダイスとアイテムの基礎組み合わせ
2. 悪い目をあえて使って相手を誘導する
3. 探索アイテムが最終儀式の触媒だと気づく

コードベースの前提:

- フロントエンドは Vanilla JS
- 主なファイルは `js/main.js`, `js/ui.js`, `js/event.js`, `js/story.js`, `js/map.js`, `js/maps/*.js`
- 章メタデータは `js/chapter_data.js`
- 再構成仕様書は `docs/gunma_rebuild_spec.md`

今回あなたにやってほしいのは「差分実装を進めるための具体的な成果物」です。
一般論ではなく、実装に使える形で出してください。

以下を出力してください。

1. `event.js` を 6章+終章に寄せるためのリファクタリング方針
   - 何を削るか
   - 何を統合するか
   - どの event_id を新設/改名すべきか

2. `story.js` と `event.js` の責務分離案
   - どちらを残すべきか
   - どちらを薄くすべきか
   - 実装を壊さず段階移行する手順

3. そのまま JS に貼れるレベルの疑似コードまたはコードスニペット
   - 新しい journey / arc 管理
   - ボス戦の「浄化条件」
   - 儀式の敬意ゲージ
   - 探索アイテムを最終儀式触媒として再利用する仕組み

4. 第1実装スプリントでやるべきタスクを、優先順位つきで 8〜12 個
   - 各タスクに対象ファイルを書くこと
   - 「先にやる理由」も短く添えること

5. 最後に、私がそのままコーディングに入れるように
   「最初の3コミット分の具体案」を書いてください
   - コミット1: 何を変えるか
   - コミット2: 何を変えるか
   - コミット3: 何を変えるか

出力ルール:
- 日本語
- 具体性優先
- 可能なら箇条書き
- ふわっとした感想は不要
- 「このコードベースならこうする」という前提で書く

最重要:
- 一本道の推進力を止めない
- 「切ない旅」と「土地への敬意」を壊さない
- 戦闘を作業にしない
```
